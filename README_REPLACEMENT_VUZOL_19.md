## 7. FACT / MODEL / FICTION / HOLD

Every theory, scene, mechanism or claim must separate:

```yaml
FACT:
  meaning: "measured / established / directly supported"

MODEL:
  meaning: "explanation / structure / hypothesis / mechanism"

FICTION:
  meaning: "story / metaphor / symbolic extension / scene logic"

HOLD:
  meaning: "unknown / unvalidated / unsafe / premature"
```

Rule:

```text
Model is not fact.
Metaphor is not proof.
Prediction is not permission.
Unknown must remain visible.
```

HOLD is not failure.

HOLD is how the system protects reality from premature certainty.

## 9. Sri Cube

The Flower gives direction.

The Sri Cube checks volume.

A scene, theory, action, technology or AI-agent step is not stable until it has six faces:

```yaml
SRI_CUBE:
  intent:
    question: "Who or what wants to move?"

  shadow:
    question: "What hidden force may distort it?"

  mechanism:
    question: "How does the movement become form?"

  constraint:
    question: "What law, boundary or Guard limits it?"

  cost:
    question: "Who or what pays the price?"

  memory:
    question: "What remains after the action?"
```

If any face is missing:

```text
verdict = HOLD / REWRITE
```

## 11. Science Theory Gate

Vuzol-19 may contain models that touch science, physics, chemistry, biology, climate, medicine, Earth, electron, molecule, body, brain, ancient structures or engineering.

Science Gate protects the repository from overclaiming.

A scientific claim must be classified as:

```text
FACT      = measured / established
MODEL     = explanatory structure
FICTION   = symbolic / narrative / speculative extension
HOLD      = not yet validated
```

To become `TEST_ALLOWED`, a claim needs:

```text
1. formula
2. prediction
3. difference from existing theory
4. experiment
5. failure condition
```

Examples:

```text
Bindu-electron as metaphor: KEEP
Bindu-electron as physics: HOLD
Bindu-electron with measurable phase shift: TEST_ALLOWED

molecule docking score: MODEL
medicine claim: HOLD until validation

weather event: FACT
climate attribution: HOLD until analysis
```

Core rule:

```text
Metaphor may guide model.
Metaphor does not prove science.
```

See:

```text
SCIENCE_THEORY_GATE.md
```

Use:

```text
templates/PRE_SCENE_RUNTIME_TEMPLATE.md

```

```text
1. Choose a scene seed.

3. Separate FACT / MODEL / FICTION / HOLD.
4. Mark Rune State.
5. Run Flower Scan.
6. Run +3 Formation and -3 Validation.
7. Check Sri Cube.
8. Check Octave Gate if the scene crosses level.
9. Produce Bindu Verdict.
10. Write scene only after verdict.
11. Run Post Scene Audit.
12. Update Memory Ledger.
```

A scene should show:

```text
body signal
pressure
shadow
Human Gate
a real choice or meaningful HOLD
consequence
memory update
```

A bad scene usually:

